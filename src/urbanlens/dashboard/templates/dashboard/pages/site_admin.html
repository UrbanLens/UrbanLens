auto_awesome Background Enrichment
{% csrf_token %}

Once an hour, proactively backfill official names, aliases, street addresses, and property/building boundaries for pins and wikis that are still missing them, prioritizing the locations most users will see. Only spends API budget left over after real traffic -- per-service limits from the API Limits page are always respected.

Hourly enrichment runs only between the start and end hours (UTC). Set both to the same value to allow enrichment at any hour; the window may wrap midnight (e.g. 22 to 4).

Share of every API limit kept in reserve for traffic spikes. With a 10% buffer and a 300-calls/30-days limit, enrichment paces itself to at most 9 calls per day and never spends past 270 in any 30-day window.

Hard cap per hourly run for each API service, even when its rate limit would allow more, so services with very generous limits are never hammered in bursts. Requests within a run are additionally staggered to stay under each service's per-minute limit.

database Storage & Uploads
{% csrf_token %}

Photo/video storage each regular user may use. Subscription roles can grant a larger quota on the Subscription Access page. Set to 0 for unlimited.

Photos larger than the maximum dimension below are downscaled on upload to save storage space. The full EXIF metadata of the original file is always saved before any conversion.

Longest edge photos are downscaled to. 1920 px keeps plenty of detail for screens while reducing the size of a modern phone photo to roughly 1/8th.

Re-encode processed uploads as WebP for additional storage savings (roughly a third smaller than JPEG at the same quality).

Whether photos uploaded by users with an active subscription are also downscaled and converted. When off, subscribers keep their original files (unless they opt into downscaling in their own settings).

Maximum size for a single photo, video, or document upload. Enforced on both the upload form and the server.

Videos taller than the maximum height below are re-encoded on upload to save storage space.

Vertical resolution videos are downscaled to. 1080 px (Full HD) keeps plenty of detail while meaningfully reducing storage for 4K phone footage.

Whether videos uploaded by users with an active subscription are also downscaled. When off, subscribers keep their original resolution (unless they opt into downscaling in their own settings).
