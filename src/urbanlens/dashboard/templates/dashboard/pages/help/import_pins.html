Google Takeout / Google My Maps

Best for saved Google Maps lists and custom Google My Maps layers. UrbanLens can read KML/KMZ and GeoJSON-style point exports.

  1. Go to Google Takeout and select the Maps data you want to export.
  2. For Google My Maps, open the map, use the menu beside the map title, and choose the export option for KML/KMZ.
  3. Download the archive, then upload the KML/KMZ file or the full ZIP if it contains supported files.
Tip: If Takeout gives you multiple files, import one layer at a time so names, descriptions, and categories are easier to verify.

Multiplottr

Use Multiplottr exports when you have a simple list of plotted points or map batches you want to preserve in UrbanLens.

  1. Open your Multiplottr map and look for export, download, or share options.
  2. Choose KML if available. If KML is not available, export a CSV with latitude and longitude columns.
  3. Upload the exported file and confirm that each row has a name plus valid coordinates.

Google Earth

Google Earth projects are usually a good fit for KML/KMZ import, especially if you grouped places into folders.

  1. Open the project or folder in Google Earth.
  2. Export or save it as KML/KMZ.
  3. Upload the file to UrbanLens. Folder names may help you decide which tags to apply after import.

GPX apps and GPS devices

Gaia GPS, CalTopo, Garmin, AllTrails, Wikiloc, Geocaching.com, and similar tools commonly export GPX. UrbanLens imports GPX waypoints as pins.

  1. Export your saved waypoints or map items as GPX. KML also works when the app offers it.
  2. If your app exports tracks and routes, add waypoints for the places you actually want as pins.
  3. Upload the GPX file and review waypoint names and notes in the preview step.

GIS portals and open data

City, county, state, federal, university, and preservation datasets often export Shapefile, GeoJSON, CSV, or ZIP archives.

  1. Find the dataset's download or export panel.
  2. Prefer GeoJSON for point datasets. Use Shapefile or ZIP when GeoJSON is not offered.
  3. For CSV exports, make sure latitude and longitude columns are included.

OpenStreetMap / Overpass Turbo

Use OSM and Overpass exports when you are importing candidate places from map data, such as historic, industrial, abandoned, or infrastructure features.

  1. Build a query in Overpass Turbo.
  2. Export the results as GeoJSON for the cleanest import, or OSM XML if you need original OSM tags.
  3. Review imported pins carefully. Public map tags can be incomplete or inaccurate.
table_chart

CSV spreadsheets

CSV is the easiest path for Airtable, Google Sheets, Excel, personal notes, scraped lists, or manually curated research.

  1. Create columns for name, latitude, longitude, and optionally description, category, tags, or source URL.
  2. Export or download the sheet as CSV.
  3. Upload the CSV and verify that coordinates are in decimal degrees.
Coordinate format: Use separate latitude and longitude columns like 42.3601 and -71.0589.

Import tips

  • Start with a small export first if you are testing a new service.
  • Use tags after import to record the source, such as google-takeout or caltopo.
  • Keep vulnerable places private and only share with trusted people.
  • If an import fails, try exporting as KML or GeoJSON, which usually preserve names and descriptions well.
