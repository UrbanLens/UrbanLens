The route drawing is backed by a standalone MarkupMap: - The check-in detail page passes markup_map_uuid + the resolved markup_json_url/markup_create_url/markup_edit_url_template for that map, plus markup_map_view_url_template so viewport changes (pan/zoom/layer) autosave onto the map. - The check-in creation page passes no uuid but sets markup_map_create_url (+ the markup_map_*_template vars consumed by createMarkupToolbar below): the first drawn annotation creates a draft MarkupMap, whose uuid lands in the hidden "markup_map" input so the create POST can link it.
