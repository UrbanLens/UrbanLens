{% comment %} Shared photo lightbox - single source of truth for every full-size photo viewer on the site (pin detail / location wiki / safety check-in galleries, Memories > Photos, the user profile page). Include once per page; open it via `window.galleryOpenLightboxItem(list, idx)`, where `list` is an array of normalized items: {url, thumbUrl, caption, author, copyright, sourceUrl, sourceName, takenAt, imageId, canRelevance, relevant, mediaSource, mediaKey, latitude, longitude} `latitude`/`longitude` (numbers, or omitted/null) draw the small location map in the side panel - see #lightbox-location below. The marker is draggable (and drags persist via `window.galleryRepositionImage`, reusing whichever gallery context - pin/wiki/safety - is on the page) only when both `imageId` and that function are present; otherwise the map renders read-only. `thumbUrl` is optional - a browser-safe fallback shown in place of `url` if the full-resolution file fails to load (e.g. a Wikimedia .tif result). Context: lightbox_id - DOM id for the (default "gallery-lightbox"). show_cover_action - whether to render the "Set as cover photo" button (only meaningful when photos belong to a single pin/location, e.g. pin detail or location wiki). cover_photo_url - POST endpoint for "Set as cover photo"; blank hides it. current_cover_image_id - the pin/wiki's current cover_photo id (if any), so the button can offer "Remove cover photo" instead of "Set as cover photo" for that image. cover_hero_key - "pin" or "wiki": which on-page cover-hero banner (see partials/js/cover-hero.js) to live-update when the cover changes - drives the "#-cover- hero" element id and the "" passed to window._coverHeroNav. Blank when the page has no such banner (e.g. safety check-ins). cover_hero_enabled - whether the banner should actually be shown once a cover exists (the wiki hero also depends on the viewer's show_wiki_cover_photos preference, which a live update must keep respecting). pin_media_relevance_url - POST endpoint for the pin Media gallery's relevant/ not-relevant marking; blank when not applicable (harmless even if set - only used for items with canRelevance true, which pages outside the pin Media gallery never set). show_attachment_points - whether to load "Shared via" (wiki/DM) info for the current item's owner (see PhotoAttachmentPointsView) - only the profile photo strip sets this; every other lightbox use leaves it off so it doesn't fire a needless request per photo. {% endcomment %} {% with lightbox_id=lightbox_id|default:"gallery-lightbox" %}
