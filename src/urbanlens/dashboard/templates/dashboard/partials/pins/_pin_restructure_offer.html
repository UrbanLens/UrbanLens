{% comment %} The single "organize this property?" suggestion, shown once per pin the first time its detail page is opened. Covers both halves together - creating a child pin per building here, and nesting existing top-level pins that stand inside this property - because they are one question about one pin's hierarchy, and two dialogs for that would be two interruptions. See controllers.pin_restructure. Reuses the shared onboarding-card markup (see ts/shared/onboarding-tour.ts and _map.scss's .page-onboarding-card rules) so it reads as the same kind of suggestion the rest of the app makes, with no styles of its own. All three actions swap this element away: Yes applies the plan (and toasts its own result), No dismisses this pin permanently, and "Don't show again" also turns the account-wide setting off. Context: pin: the pin being viewed. building_count / sample_names: buildings here with no pin yet. nestable_count / nestable: the owner's top-level pins inside this property (nestable is capped for display; nestable_count is the real total). {% endcomment %}
