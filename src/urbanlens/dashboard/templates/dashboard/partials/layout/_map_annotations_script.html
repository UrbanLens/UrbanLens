{% comment %} Shared map annotations script - markup drawing/editing, the unified detail-pin side panel, the typed boundary editor (+ its context menu), and the Details/Photos layers list panel. Used identically by the pin detail page and the Location wiki page; a fix here lands on both. Must be included after the page has already created its Leaflet `map` and opened `
` inside a `.map-wrapper` (see index.html/wiki.html), and after `_markup_engine.html` (defines window.MarkupEngine) and toastr are loaded. Required include context (pass via `{% include ... with %}`): map_center_lat, map_center_lng - numeric center for the map view markup_json_url, markup_create_url, markup_edit_url_template detail_pins_json_url, detail_pin_create_url, detail_pin_edit_url_template boundary_url - single endpoint, GET (load) + POST (save) photo_gallery_json_url Optional: main_marker_owner_uuid - if set (a Pin's UUID), the map's center marker is draggable and PATCHes /dashboard/rest/pins// with its new position on drag; otherwise a plain, non-draggable marker is shown (Locations don't support dragging their canonical coordinates - those are edited via "Suggest edits" instead). default_map_view - 'satellite' | 'topographic' | 'street' (default: satellite) openweathermap_api_key - if unset, the Rain/Clouds overlay layers are omitted Inherited from the parent page's own context (not re-passed): pin_type_choices, detail_pin_icon_choices, color_choices, markup_fill_color, markup_fill_opacity, markup_border_color, markup_border_opacity. {% endcomment %}
